Built with Claude Code · Most Inventive Use

We didn't just build a routing agent.
We taught Claude Code to guard it.

Two slash command skills

/route-lead

Type a lead in plain English inside Claude Code. The full pipeline runs. You get category, tier, rep, confidence score, and reasoning chain — inline, in your session.

/run-scorecard

Run the eval harness and see the full stratified scorecard without leaving the tool. dry-run validates all cases with no API calls.

Two hooks in .claude/settings.json

PreToolUse — eval guard

Before any edit to agent logic, Claude Code runs the eval dry-run automatically. If test cases break, the edit is blocked — in the same session, before CI.

PostToolUse — threshold reminder

After editing config.py, Claude Code reminds you to update MANDATE.md and SCORECARD.md. The policy doc stays in sync with the code.

The safety loop is tighter than the deployment loop.

The Mandate · Legal-Ready Contract

The agent has a written contract.
It knows exactly where its authority ends.

Decides alone

Route

confidence ≥ 0.75 · rep available · no adversarial signals

Disqualify

spam, competitor probe, prompt injection detected

Request more info

confidence < 0.50 and no enterprise signals

Deliberately NOT automated

  • 🔒 Frozen / legal-hold accounts — always escalate
  • 📧 Sending emails — agent drafts, human sends
  • ⚖️ OFAC / sanctions signals — always escalate
  • 🏢 C-level, board, legal-direct routing — hook-blocked
  • 🔁 Re-classifying to inflate confidence score
The Brake · Two-Layer Safety

Hard stop + slow stop.
A prompt can't override a hook.

⛔ PreToolUse Hook — hard stop
